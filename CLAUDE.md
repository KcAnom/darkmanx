## Source of truth vs. mirrors

Edit these (source of truth):
- `skills/*/SKILL.md`, `skills/*/README.md`
- `agents/*.md`
- `src/rules/*.md`
- `src/hooks/*.js`
- `src/tools/*.js`
- `bin/install.js`, `bin/lib/*.js`
- `src/mcp-servers/darkman-x-shrink/*`
- `src/plugins/opencode/*`
- `commands/darkman-x*.md` + matching `.toml` (Claude Code slash commands)
- `.pi/extensions/darkman-x.ts`, `.pi/prompts/*.md`, `.pi/README.md`, `.pi/settings.json`, `.pi/darkman-x-pi.env.example` (Pi coding-agent sidecar — see `.pi/README.md`'s own "Source of truth" section)

Never hand-edit these:
- `plugins/darkman-x/agents/*` — copy of `agents/*`, CI-generated (regenerated by `.github/workflows/sync-skill.yml` on every push to `main` that touches `skills/**` or `agents/**`)
- `plugins/darkman-x/skills/*` — copy of selected skills + compress scripts, CI-generated
- `dist/darkman-x.skill` — zip build artifact of `skills/darkman-x/`, gitignored
- `.pi/skills/*` — symlinks into `skills/*` (edit the target under `skills/`, never the link)

If you find yourself editing anything under `plugins/darkman-x/`, stop — go edit the corresponding file under `agents/` or `skills/` instead and let CI resync.

**`.pi/` is real, tracked, and live** — not a mirror, not a sandbox. It's Pi's own native implementation of darkman-x (extension API, not markdown templates), git-tracked and pushed to this repo's `main` like everything else. The user runs Pi directly in this same local checkout, so a Pi session can commit/push to `main` concurrently with a Claude Code session — re-check `git log`/`git status` rather than trusting an earlier snapshot in a long conversation. Keep `.pi/` in parity with the Claude Code side: same command names, same confirmation wording, same status fields (see `commands/darkman-x*.md` + `.toml` vs `.pi/prompts/darkman-x*.md` + `.pi/extensions/darkman-x.ts`'s `registerCommand` calls).

## Rules for agents working in this repo

1. **Hooks must silent-fail.** Never let a hook throw or block session start — wrap risky work in try/catch and no-op on failure.
2. **All flag/config writes go through `safeWriteFlag`** (`src/hooks/darkman-x-config.js`) — it defends against symlink clobbering. Don't call `fs.writeFileSync` directly on flag paths.
3. **`settings.json` is read/written only via the JSONC-safe helpers** in `bin/lib/settings.js` (`stripJsonComments`, `stripTrailingCommas`, `validateHookFields`) — Claude's settings schema is Zod-strict, malformed hooks must be dropped, not written.
4. **Command formats stay aligned across all three surfaces** — every `commands/darkman-x*.md` has a matching `.toml` with the same behavior described, and (where a Pi-native equivalent exists) a matching `.pi/prompts/darkman-x*.md` fallback describing the same command/subcommands/confirmation shape.
5. **Independent modes (commit/review/compress) are one-shots** — they must save the previous prose mode and restore it, never permanently displacing full/lite/ultra.
6. **No fabricated benchmark numbers** — see `docs/HONEST-NUMBERS.md`. If you don't have a real measurement, label it clearly as a placeholder.
7. **Compress style, never language or meaning** — `darkman-x-compress` and the MCP shrink proxy must never mangle code fences, inline code, URLs, or paths.
8. **No caveman/prehistoric voice.** This is an intentional rebrand — Darkman X / DMX energy only. See `RECREATION-PROMPT.md` for the full brand-voice rules.
9. **SFX clips never get committed.** `src/tools/darkman-x-sfx.js` plays audio from an out-of-tree directory (`~/.config/darkman-x/sfx/` by default) — never add clip files to this repo, this repo is public, and the clips are copyrighted. Triggering is model-judgment only (no hardcoded event→clip map) — don't build one.

Full architecture and file-by-file build spec: [`RECREATION-PROMPT.md`](./RECREATION-PROMPT.md).
